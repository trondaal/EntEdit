1. Getting started

EntEdit requires a connection to a SPARQL-compatible graph database set up for bibliographic descriptions using the RDA vocabulary. On first launch, a configuration wizard will guide you through the setup.

1.1 Configuration wizard

When you open EntEdit for the first time, a four-step wizard appears:

  1. Database connection — Enter the URL of your SPARQL endpoint, and optionally a username and password if the endpoint requires authentication.
  2. Test connection — EntEdit verifies that it can reach the database and query it.
  3. Language — Choose your preferred language (currently English or Norwegian).
  4. Settings — Configure optional save warnings. You can enable warnings that remind you when you are about to save an entity with an auto-generated URI or an auto-generated label. These warnings are off by default — see Auto-generation of URIs and labels for details.

You can change these settings at any time by clicking the endpoint chip or the settings icon in the header bar.

1.2 Connecting to a database

There are several ways to get a database up and running:

Option Endpoint URL Description
Demo database http://dijon.idi.ntnu.no:8080/repositories/EntEdit A shared demo instance pre-loaded with sample data. Good for trying out the application without any local setup.
Docker (recommended) http://localhost/graphdb/repositories/EntEdit Run the full stack locally with docker compose up -d. This starts both the web application and a GraphDB instance with sample data. The app is then available at http://localhost/entedit/.
Your own database (varies) Set up a GraphDB or other SPARQL-compatible database with your own data and ontology. See the Database Setup Guide for detailed instructions.
Tip: If you are a student in a course, your instructor will tell you which endpoint URL to use. In most cases you will be using Docker to run the application and the database.

1.3 Language selection

The language setting (available in the header bar and in the configuration wizard) affects two things:

  • User interface — All buttons, labels, and messages in the application are shown in the selected language.
  • Database content — When the database contains labels in multiple languages (for classes, properties, and controlled values), EntEdit will use labels tagged with the selected language. If a label is not available in the selected language, it falls back to the English or to values with no language tag.

EntEdit currently supports English and Norwegian. You can switch language at any time without losing your work.

2. What is entity-oriented cataloguing?

Traditional library cataloguing focuses on describing publications — the physical or digital items you find on a shelf or online. Entity-oriented cataloguing takes a different approach: it describes the world of intellectual and artistic endavour as a network of distinct entities and the relationships between them.

Instead of creating one record that mixes information about a novel, its author, a particular translation, and a specific printed edition, entity-oriented cataloguing separates these into distinct entities — each described individually and linked together. This makes it possible to clearly represent facts like:

  • A novel exists in multiple translations, each with their own translator.
  • Editions that share the same text but differ in publisher, year, or format.
  • A film is based on a novel — they are related but considered different creative works.

EntEdit implements the IFLA Library Reference Model (IFLA-LRM) using the RDA (Resource Description and Access) vocabulary. Both are well established international standards for library metadata.

3. The WEMI model

At the heart of IFLA-LRM is the WEMI model, which organises bibliographic information into four levels of abstraction:

Work Intellectual creation
→
Expression Realisation
→
Manifestation Embodiment
→
Item Individual copy

Each level captures a distinct aspect of a bibliographic resource:

Entity What it represents Example
Work The abstract intellectual or artistic creation — the concept of "this novel" or "this symphony" independent of any particular version or edition. Cormac McCarthy's novel The Road
Expression A specific intellectual realisation of a Work. Each translation, performance, or arrangement is a separate Expression. The original English text of The Road; a Norwegian translation; an audiobook narration
Manifestation A physical or digital embodiment of an Expression — a particular publication, pressing, or file. The 2009 Picador paperback (ISBN 978-0-330-46846-6)
Item A single copy or instance of a Manifestation — a specific book on a specific shelf. The copy in Oslo Public Library with barcode 03010712345
Why separate these levels? Think of it this way: a Work can have many Expressions (translations, abridgements, performances). Each Expression can appear in many Manifestations (hardback, paperback, e-book, different publishers). This structure lets us say things like "all Norwegian translations of Hunger by Hamsun" or "every edition of the English text" — queries that are very difficult with flat catalogue records.

In addition to the WEMI entities, we have:

Entity What it represents Example
Agent A person, organisation, or family who has a relationship to a Work, Expression, or Manifestation — authors, translators, publishers, performers, etc. Cormac McCarthy (author), Picador (publisher)
Note on Agent: In the full RDA model, Agent is a supertype with more specific subtypes such as Person, Collective Agent, and Family. EntEdit uses only the general Agent class as a simplification — since we only use properties that are common to all agents (such as name and dates), there is no need to distinguish between persons and organisations.

4. Using EntEdit

4.1 The Entity Browser

The Entity Browser is the main interface, organised in three panels:

Position Panel Purpose
Left Classes Lists the entity types (Work, Expression, Manifestation, Item, Agent, etc.). Click a class to see its instances.
Middle Entities Shows all entities of the selected class. Use the filter field to narrow the list. Scroll down to load more.
Right Editor Displays and edits the properties of the selected entity.

4.2 Creating and editing entities

Creating a new entity

  1. Select a class in the left panel (e.g. Work).
  2. If you have not selected an existing entity, you can edit a new entity directly.
  3. If an existing entity is selected, click the ⋮ (more actions) menu in the editor panel and choose New to create a new entity of the selected class.
  4. Fill in text metadata using the Add dropdown under "Text metadata" — this is where the descriptive cataloguing data goes.
  5. Add controlled values, relationships, and agent links from the respective sections. In these fields, you enter values by selecting from the list that is displayed.
  6. When done, click Save (or press Ctrl+S).

You do not need to provide a URI or a label — both are auto-generated if left blank (see Auto-generation of URIs and labels). If you want to set these manually, use the identifier field for the URI and the label editor for the label.

Editing an existing entity

  1. Select the entity in the middle panel.
  2. Click the Edit button in the editor panel.
  3. Make changes to properties, labels, or relationships.
  4. Click Save to persist, or Cancel to discard changes.

Linking entities together

Relationships are added through dedicated sections in the editor:

  • WEMI relationships — Link a Work to its Expressions, an Expression to its Manifestations, etc.
  • Related Agents — Link an entity to its authors, translators, publishers, etc.
  • Related Works / Expressions / Manifestations — Link entities of the same type (e.g. a film adaptation Work to the novel Work it is based on).

To add a relationship:

  1. Make sure the entity is in edit mode.
  2. Open the relevant relationship section (e.g. "Related Agents").
  3. Use the Add dropdown to select the type of relationship.
  4. An entity picker appears — filter and select the target entity.
  5. The relationship is added. Click Save to persist.
Tip: You may need to create the target entity first. For example, before linking a Work to its author, make sure the Agent entity for that author already exists in the database.

4.3 More actions menu

The ⋮ (more actions) button in the top-right corner of the entity editor provides access to several additional features:

Action Description
New Clear the editor and start creating a new entity of the currently selected class.
Graph Open the entity in the GraphDB Workbench graph visualization, where you can explore its relationships visually in a new browser tab.
Turtle RDF Export the entity's RDF data in Turtle format — see RDF and Turtle export.
Delete Permanently delete the entity and all its statements from the database — see below.

Exporting an entity as Turtle RDF

The Turtle RDF menu option exports the current entity's data as Turtle — see RDF and Turtle export for a full description of Turtle and the export features.

Note: You must save any unsaved changes before exporting. The export option is disabled when the entity has been modified — save first, then export.

Deleting an entity

The Delete option permanently removes the entity and all its statements (both outgoing and incoming) from the database. A confirmation dialog is shown before the deletion is carried out. This action cannot be undone.

Important: Deleting an entity also removes all references to it from other entities. For example, deleting an Agent will remove the authorship links from any Works that referenced that Agent.

Reordering multi-value properties

When a property has more than one value (e.g. multiple notes on a Manifestation), you can reorder them in edit mode. Drag handles and up/down arrow buttons appear next to each value, allowing you to set the preferred display order by dragging or clicking the arrows.

4.4 The Search interface

The Search tab offers full-text search across the database with two modes:

  • Expression search — Finds Expressions (and shows their related Work, agents, and Manifestations). Expand a result to see all Manifestations of that Expression.
  • Manifestation search — Finds Manifestations directly by their title, publisher, identifiers, or other metadata.

Search results display agents, metadata, and relationships in a compact format, letting you quickly survey what is already catalogued.

5. RDF, URIs, and labels

5.1 What is RDF?

All data in EntEdit is stored as RDF (Resource Description Framework) — the standard data model of the semantic web. In RDF, every piece of information is expressed as a triple: a subject, a predicate, and an object.

An RDF triple

<The Road>   <has author>   <McCarthy, Cormac> .

subject → predicate → object

The subject and predicate are always identified by URIs (Uniform Resource Identifiers) — globally unique web addresses that name things unambiguously. The object can be either another URI (linking to another entity) or a literal value such as a text string, number, or date.

When you create an entity in EntEdit and fill in its properties, you are building a collection of RDF triples. For example, a Work entity with a title, a language, and a link to its author becomes three triples — each connecting the entity URI to a value via a property URI.

5.2 URIs and labels

Every entity in RDF is identified by a URI. This is its unique identifier — no two entities share the same URI. In a production setting, you would typically use established identifiers from authority services such as VIAF (e.g. http://viaf.org/viaf/29558386 for Cormac McCarthy) or from your institution's own namespace, but for learning and experimentation, EntEdit can generate URIs automatically.

In addition to the URI, every entity in EntEdit has a label (rdfs:label). This is not an RDA cataloguing property — it is a general semantic web convention used to give entities a short, readable name. In EntEdit, labels are what you see in the entity lists and search results. Without a label, an entity would only be shown by its URI, which is not very helpful.

Labels can be added in multiple languages. They are separate from the formal RDA properties (such as title of work or access point name) that carry the actual cataloguing data.

5.3 Auto-generation of URIs and labels

EntEdit is designed to let you focus on the cataloguing data. By default, both the entity URI and the label are generated automatically when you save:

Field What happens when left blank
URI A unique URI is generated based on a timestamp (e.g. http://example.org/entity-1712345678901). This is suitable for learning and experimentation. In a production setting you would typically provide a meaningful URI from your institution's namespace.
Label The value of the entity's primary property (the first text metadata field, e.g. title of work or access point name) is used as an untagged rdfs:label.

This means you can create a complete entity by just filling in the cataloguing properties and clicking Save — the URI and label are taken care of for you.

You can always override the auto-generated values using the ⋮ (more actions) menu in the entity editor. Use the URI option to enter a pre-existing identifier (e.g. a VIAF URI for an Agent), and the label option to create custom, language-tagged labels.

Custom labels can be more informative than auto-generated ones. Example labels:

  • Works: Title followed by the author in parentheses, e.g. "The Road (McCarthy, Cormac)"
  • Expressions: Title followed by language and content type in parentheses, e.g. "The road (english text)"
  • Manifestations: A brief citation-style string, e.g. based on ISBD "The road / Cormac McCarthy. — Picador, 2009"
  • Agents: The name, e.g. "McCarthy, Cormac"

The label editor

Click the label icon in the editor header to open the label editor. It opens as a draggable dialog that you can reposition while still interacting with the entity editor behind it.

Each label is shown as an editable row with a text field and a language dropdown. You can:

  • Type directly into any label row — all rows are always editable.
  • Select a language tag (e.g. EN, NO) from the dropdown, or leave it empty for no language tag.
  • Click the Add button (or press Enter on the last row) to add a new label row.
  • Click the trash icon to remove a label.
  • Click Apply to save your changes, or Cancel to discard them.

Empty rows are automatically removed when you apply.

Save warnings

If you want to be reminded to enter URIs and labels manually, you can enable save warnings in the configuration wizard (step 4) or in the settings. When enabled, EntEdit will show a confirmation dialog before saving an entity with an auto-generated URI or label, giving you the chance to go back and enter them yourself.

These warnings are useful when you want to ensure high-quality RDF data with established URIs and descriptive, language-tagged labels.

5.4 Turtle format

Turtle (.ttl) is a compact, human-readable way of writing RDF triples. It uses prefixes to shorten long URIs, making the data easier to read. Here is an example of how a Work might look in Turtle: 

@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix rda:  <http://rdaregistry.info/Elements/w/> .

<http://example.org/entity-123>
    a <http://rdaregistry.info/Elements/c/C10001> ;
    rdfs:label "The Road" ;
    rda:P10088 "The Road" ;
    rda:P10436 <http://example.org/entity-456> .

In this notation, a means "is of type", the semicolons separate properties of the same entity, and angle brackets enclose full URIs. Prefixed names like rdfs:label expand to full URIs using the prefix declarations at the top.

5.5 Exporting data as Turtle

EntEdit offers two ways to export data as Turtle RDF:

Single-entity export

From the entity editor, open the ⋮ (more actions) menu and choose Turtle RDF. This exports the currently selected entity with all its properties and relationships. A dialog opens showing the formatted Turtle text, with two options:

  • Download .ttl — saves the entity as a Turtle file.
  • Copy to Clipboard — copies the Turtle text so you can paste it elsewhere.

The entity must be saved before exporting — the menu option is disabled when there are unsaved changes.

Bulk export

The Download button in the application header lets you export multiple entity types at once. A dialog shows all available classes with entity counts. Select the classes you want to include (or use "Select All") and click Export. The result is a single Turtle file containing all entities of the selected types.

Tip: Turtle files can be opened in any text editor or loaded into other RDF tools and triple stores. This makes the export useful for backups, sharing data, and interoperability with other systems.

6. Example 1 — The Road by Cormac McCarthy

Let us walk through cataloguing Cormac McCarthy's The Road — a post-apocalyptic novel published in 2006, later adapted into a film in 2009. This example demonstrates the core WEMI chain and how to express an adaptation relationship between two Works.

Here is what our entity structure will look like when we are done:

Agent: McCarthy, Cormac (1933–2023)
│
├── author of ──▸ Work: The Road
│ │
│ ├── has expression ──▸ Expression: The Road (English text)
│ │ │
│ │ └── has manifestation ──▸ Manifestation: Picador, 2009
│ │ │
│ │ └── has item ──▸ Item: Oslo Public Library copy
│ │
│ └── adapted as ──▸ Work: The Road (Film, 2009)
│
└── author of ──▸ Work: No Country for Old Men
⋮

6.1 Creating the Agent

We start by creating the author, since we will link to this Agent from the Work.

Agent McCarthy, Cormac

Property Value
Label McCarthy, Cormac
Access point (name) McCarthy, Cormac
Dates 1933–2023
  1. In the Entity Browser, select Agent in the classes panel.
  2. Click ⋮ → New in the editor.
  3. Click the label icon in the editor header and add the label "McCarthy, Cormac" (so the Agent is recognisable in lists).
  4. Under Text metadata, use Add to add the RDA properties: the access point name and the dates associated with the agent.
  5. Click Save.

6.2 Creating the Work

The Work represents the novel as an abstract intellectual creation — independent of any particular edition or format.

Work The Road

Property Value
Label The Road (McCarthy, Cormac)
Title of work The Road
Author → Agent: McCarthy, Cormac
  1. Select Work in the classes panel, then ⋮ → New.
  2. Add a label, e.g. "The Road (McCarthy, Cormac)". Including the author in parentheses is a common convention to help distinguish Works with the same title in entity lists.
  3. Add the RDA text property Title of work with value "The Road".
  4. In the Related Agents section, add the relationship "has author" and select the Agent "McCarthy, Cormac" from the entity picker.
  5. Click Save.
Label vs. title of work: Remember that the label is just a display name for use in lists — it is not part of the RDA description. The title of work is the formal RDA property that records the actual title. That is why we add "The Road" as a text metadata property even though the label already contains it.

6.3 Creating the Expression

An Expression represents a specific intellectual realisation of the Work. For a novel, the original English text is one Expression. A Norwegian translation would be a different Expression of the same Work.

Expression The Road — English text

Property Value
Label The Road (McCarthy, Cormac)
Title of expression The Road
Language English
Content type text
Is expression of → Work: The Road
  1. Select Expression, then ⋮ → New.
  2. Add a label (e.g. "The Road (McCarthy, Cormac)") and the RDA property title of expression.
  3. Under Controlled values, add Language → English and Content type → text.
  4. Under WEMI relationships, add "is expression of" and select the Work "The Road".
  5. Click Save.

6.4 Creating the Manifestation

A Manifestation is a specific published edition. This is where we record the physical details familiar from traditional cataloguing: publisher, date, extent, ISBN.

Manifestation The Road — Picador, 2009

Property Value
Label The road / Cormac McCarthy. — Picador, 2009
Title proper The road
Statement of responsibility Cormac McCarthy
Place of publication London
Publisher name Picador
Date of publication 2009
Extent 307 p., 20 cm.
Identifier ISBN 978-0-330-46846-6 (pbk.)
Note Film tie-in.
Note Originally published: 2006.
Media type unmediated
Carrier type volume
Has expression manifested → Expression: The Road (English text)
  1. Select Manifestation, then ⋮ → New.
  2. Add a label. A common convention for manifestation labels is: Title / Responsibility. — Publisher, Date.
  3. Add the RDA text metadata: title proper, statement of responsibility, publication details, extent, identifiers, and notes.
  4. Under Controlled values, set media type and carrier type.
  5. Under WEMI relationships, add "has expression manifested" and select the Expression you created.
  6. Click Save.
Tip: A Manifestation can have multiple notes and multiple identifiers. Use the Add dropdown repeatedly to add more than one value for the same property.

6.5 Creating an Item

An Item represents a specific copy of a Manifestation — a particular book on a particular shelf. Items are useful when you need to track individual copies, for example in a library collection.

Item Oslo Public Library copy

Property Value
Identifier 03010712345
Note Copy in good condition.
Is item of → Manifestation: The road / Cormac McCarthy. — Picador, 2009
  1. Select Item in the classes panel, then ⋮ → New.
  2. Under Text metadata, add the identifier (e.g. a barcode or accession number) and optionally a note.
  3. Under WEMI relationships, add "is item of" and select the Manifestation "The road / Cormac McCarthy. — Picador, 2009".
  4. Click Save.
When to use Items: Not every cataloguing scenario requires Item entities. Items are most useful when you need to track individual copies for circulation, inventory, or provenance purposes. For many purposes, the Manifestation level is sufficient.

6.6 Adding the film adaptation

The Road was adapted into a film directed by John Hillcoat in 2009. In the WEMI model, this is a new Work — a distinct intellectual creation that is related to the original novel through an adaptation relationship.

Work The Road (Film, 2009)

Property Value
Label Road (Motion picture : 2009)
Title of work The Road
Is motion picture adaptation of → Work: The Road (McCarthy, Cormac)
  1. Create a new Work for the film.
  2. Under Related Works, add the relationship "is motion picture adaptation of" and select the novel's Work entity.
  3. Click Save.

This creates an explicit link showing that the film derives from the novel, while keeping them as separate creative works with their own Expressions and Manifestations (e.g. a DVD release would be a Manifestation of the film's Expression).

Adaptation vs. Expression: A translation is a new Expression of the same Work — the intellectual content is essentially the same, just in a different language. A film adaptation is a new Work — it is a different intellectual creation inspired by the original. This distinction is fundamental to the WEMI model.

7. Example 2 — Sult by Knut Hamsun

Knut Hamsun's Sult (1890) is an excellent example for understanding how translations and multiple editions are modelled. The novel has been translated into English under the title Hunger by several different translators, and published in numerous editions in both Norwegian and English.

Agent: Hamsun, Knut (1859–1952)
│
└── author of ──▸ Work: Sult
│
├── has expression ──▸ Expression: Sult (Norwegian text)
│ ├── Manifestation: Gyldendal, 2001
│ ├── Manifestation: Gyldendal, 2009
│ └── Manifestation: ⋮ (more editions)
│
├── has expression ──▸ Expression: Hunger (English, transl. Lyngstad)
│ ├── Manifestation: Penguin, 1998
│ └── Manifestation: Canongate, 2006
│
└── has expression ──▸ Expression: Hunger (English, transl. Bly)
└── Manifestation: Farrar, Straus & Giroux, 1967

7.1 Translations as separate Expressions

Each translation of Sult is a separate Expression because the translator creates a new intellectual realisation of the Work. Even two English translations are different Expressions if they are by different translators:

One Work, multiple Expressions

Entity Description
Work Sult — the abstract novel by Hamsun
Expression 1 The original Norwegian text (language: Norwegian)
Expression 2 English translation by Sverre Lyngstad (language: English, translator: Lyngstad)
Expression 3 English translation by Robert Bly (language: English, translator: Bly)

To create a translated Expression:

  1. Create a new Expression.
  2. Set the language to the target language (e.g. English).
  3. Set the title of expression to the translated title (e.g. "Hunger").
  4. Link it to the Work via "is expression of" → Sult.
  5. Under Related Agents, add the translator with the relationship "has translator".
  6. Click Save.
Key insight: The author (Hamsun) is linked at the Work level — he is the creator of the intellectual content. The translator is linked at the Expression level — they created this specific realisation. This separation means we only record Hamsun's authorship once, not on every edition.

7.2 Multiple editions as Manifestations

Different published editions of the same text are separate Manifestations of the same Expression. For example, the Norwegian text of Sult has appeared in dozens of editions since 1890, each with a different publisher, date, format, or ISBN — but they all share the same Expression.

One Expression, multiple Manifestations

Manifestation Publisher Date Format
Sult (hardback) Gyldendal 2001 277 p., volume
Sult (audiobook) Lydbokforlaget 2009 audio disc
Sult (e-book) Gyldendal 2015 online resource

All of these share the same Expression (Norwegian text by Hamsun) but differ in how that text is embodied.

Tip: An audiobook narration is often considered a new Expression (since the narrator contributes a creative performance) rather than just a new Manifestation. Use your judgement: if the intellectual content has been significantly transformed, it may warrant a separate Expression.

8. Example 3 — Agatha Christie 1960s Omnibus

The previous examples showed one Expression per Manifestation. But the WEMI model also supports the reverse: a single Manifestation that embodies multiple Expressions. This is common for anthologies, omnibus editions, and collected works.

The Agatha Christie 1960s Omnibus (HarperCollins, 2006) collects four of Christie's novels from the late 1960s and early 1970s into one volume. Each novel is a separate Work with its own Expression, and the omnibus Manifestation links to all four Expressions.

8.1 One Manifestation, multiple Expressions

Here is the entity structure for the omnibus:

Agent: Christie, Agatha (1890–1976)
│
├── author of ──▸ Work: Endless Night
│ └── has expression ──▸ Expression: Endless Night (English text)
│
├── author of ──▸ Work: By the Pricking of My Thumbs
│ └── has expression ──▸ Expression: By the Pricking… (English text)
│
├── author of ──▸ Work: Passenger to Frankfurt
│ └── has expression ──▸ Expression: Passenger to Frankfurt (Eng. text)
│
└── author of ──▸ Work: Postern of Fate
└── has expression ──▸ Expression: Postern of Fate (English text)

All four Expressions ──▸ has manifestation ──▸ Manifestation: Agatha Christie 1960s Omnibus
Key insight: In the previous examples each Manifestation linked to a single Expression. Here, one Manifestation links to four Expressions — one for each novel contained in the omnibus. The relationship "has expression manifested" can be used multiple times on the same Manifestation. This is the many-to-many nature of the Expression–Manifestation relationship in WEMI.

8.2 Creating the omnibus step by step

For this example we assume you have already created the Agent for Agatha Christie. If not, create the Agent first (as in Example 1).

Step 1 — Create the four Works

Each novel is its own Work. Create four Work entities:

Four Works by Agatha Christie

Work label Title of work
Endless Night (Christie, Agatha) Endless Night
By the Pricking of My Thumbs (Christie, Agatha) By the Pricking of My Thumbs
Passenger to Frankfurt (Christie, Agatha) Passenger to Frankfurt
Postern of Fate (Christie, Agatha) Postern of Fate

For each Work, link it to the Agent "Christie, Agatha" via Related Agents → "has author".

Step 2 — Create the four Expressions

Each Work has one Expression representing the original English text. Create four Expression entities, each with:

  • A label, e.g. "Endless Night (english text)"
  • The title of expression matching the novel title
  • Language: English
  • Content type: text
  • A WEMI link "is expression of" pointing to the corresponding Work

Step 3 — Create the omnibus Manifestation

Now create the Manifestation for the omnibus edition itself:

Manifestation Agatha Christie 1960s Omnibus

Property Value
Label Agatha Christie 1960s omnibus. — HarperCollins, 2006
Title proper Agatha Christie 1960s omnibus
Statement of responsibility Agatha Christie
Publisher name HarperCollins
Date of publication 2006
Extent 784 pages, 12.9 x 4.9 x 19.8 cm
Media type unmediated
Carrier type volume

Step 4 — Link all four Expressions

This is the crucial step that distinguishes the omnibus from a regular single-work Manifestation. Under WEMI relationships, add "has expression manifested" four times — once for each Expression:

  1. Add "has expression manifested" → Endless Night (english text)
  2. Add "has expression manifested" → By the Pricking of My Thumbs (english text)
  3. Add "has expression manifested" → Passenger to Frankfurt (english text)
  4. Add "has expression manifested" → Postern of Fate (english text)

Click Save to persist the Manifestation.

Tip: The order of the "has expression manifested" links reflects the order the novels appear in the omnibus. You can reorder them using drag-and-drop or the up/down arrows in edit mode.
Note on shared Expressions: If any of these novels had already been catalogued as standalone editions (e.g. a separate paperback of Endless Night), you would reuse the same Expression entity. Both the standalone Manifestation and the omnibus Manifestation would link to it. This is the power of the WEMI model — an Expression can be embodied in many Manifestations, and a Manifestation can embody many Expressions.

9. Practice exercises

Try these exercises to practice what you have learned. Start with the simpler tasks and work your way up.

Exercise A — A simple book

Catalogue a book you have on your desk or find one in a library catalogue. Create:

  1. An Agent for the author.
  2. A Work with the title and an author link.
  3. An Expression with language and content type, linked to the Work.
  4. A Manifestation with publication details, linked to the Expression.
  5. (Optional) An Item representing a specific copy — add an identifier and link it to the Manifestation.

Exercise B — A translated novel

Choose a novel that has been translated. Catalogue it following the Sult pattern:

  1. Create the Work and the Agent for the original author.
  2. Create two Expressions — the original language and one translation.
  3. Create an Agent for the translator and link it to the translated Expression.
  4. Create at least one Manifestation for each Expression.

Exercise C — A film adaptation

Choose a novel that has been adapted into a film. Following the The Road pattern:

  1. Catalogue the novel as Work → Expression → Manifestation.
  2. Create a second Work for the film adaptation.
  3. Link the film Work to the novel Work with "is motion picture adaptation of".
  4. Create an Expression and Manifestation for the film (e.g. a DVD or streaming release).

Exercise D — Collected works

Advanced: Catalogue a collected works volume (e.g. "The Complete Short Stories of…"). Think about:

  • Is this a new Work, or an Expression of an existing Work?
  • How do you represent the individual stories and their relationship to the collection?
  • What role does the editor play?

10. Key concepts reference

WEMI relationship summary

From Relationship To
Work has expression Expression
Expression is expression of Work
Expression has manifestation Manifestation
Manifestation has expression manifested Expression
Manifestation has item of manifestation Item
Item is item of Manifestation

Common agent relationships

Level Relationship Example
Work has author The novelist, the composer, the playwright
Expression has translator The person who translated the text
Expression has performer A musician performing a musical work
Expression has narrator The narrator of an audiobook
Manifestation has publisher Gyldendal, Picador, Penguin

When to create a new Work vs. a new Expression

Scenario Entity type Reasoning
Film adaptation New Work Substantially different creative creation
Stage adaptation New Work New artistic interpretation with different medium
Critique or study of a work New Work Original intellectual creation about another Work
Translation New Expression Same intellectual content, different language
Abridgement New Expression Same Work, modified text
Audiobook narration New Expression Performance adds creative content
New edition, same text New Manifestation Same Expression, different publication
Remember: The WEMI model is a way of thinking about bibliographic resources. In practice, there will be borderline cases. The important thing is to be consistent and to think carefully about what aspect of the resource you are describing at each level.
